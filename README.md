# Cover Battle — 岩と森の 4 vs 4 バトル

ブラウザだけで遊べる、見下ろし型の 4 対 4 タクティカルバトルです。
岩を盾にし、森に隠れながら、敵チームを全滅させてステージを攻略しよう。
（スプラトゥーンのような「地形を使った撃ち合い」をシンプルにしたゲームです）

## 遊び方

`index.html` をブラウザで開くだけ。インストールやビルドは不要です。

```bash
# そのままダブルクリックで開くか、ローカルサーバーでもOK
python3 -m http.server 8000
# → http://localhost:8000 を開く
```

## 操作

| 操作 | キー |
|---|---|
| 移動 | `W` `A` `S` `D` または 矢印キー |
| 照準 | マウス |
| 射撃 | 左クリック または スペース |
| 爆弾設置 | `E`（または `Q`） |
| ダイナマイト設置 | `X` |
| ロックオン 切替 | `R` |
| ロック対象 変更 | `Tab` |
| 武器選択 | `1` `2` `3` `4`（直接） / `F`（循環） |

### 武器

| 武器 | 特徴 |
|---|---|
| ライフル | 標準。10発/リロード3秒のバランス型 |
| スナイパー | 高威力・低連射・高弾速・長射程・装弾5発 |
| ショットガン | 1発で6粒拡散。近距離特化・射程短め |
| サブマシンガン | 超連射・装弾25発・威力低め・やや拡散 |

### スマホ操作

左の**バーチャルスティック**で移動（**斜め移動OK**・アナログ）、右のボタンで **撃／💣爆弾／🔒ロック／⇄対象変更**。タッチ端末では自動でロックオンON。

### 弾薬とリロード（緊迫感）

マガジンは **10発**。撃ち切ると **3秒のリロード**に入り、その間は撃てません。撃ちっぱなしにせず、残弾と間合いを管理するのが鍵。AIも同じルールで戦います（リロードの隙が攻めどき）。

## ルール

- **青チーム（4人）** を率いて **赤チーム（4人）** と戦う。あなたは青の1人を操作し、残りの味方3人はAIが担当します。
- 相手チームを **全滅** させるとステージクリア。味方が全滅すると敗北。
- ステージは全3面。クリアするごとに敵が強くなります。

## 地形

| 地形 | 効果 |
|---|---|
| 🪨 **岩** | 弾と移動をブロックする盾。撃ち続けると **破壊でき**、壊すと **アイテム** が出ることがある。 |
| ⛰️ **山** | 壊せない遮蔽。弾・視線・移動をブロックする。 |
| 🌲 **森** | 中に隠れると **敵からは見えなくなり狙われない**（味方と自分には見える）。敵が一定距離まで近づくと見つかる。 |
| 🌊 **川** | 通れるが **移動が遅くなる**。渡るときが危険。 |
| 🏰 **砦（自陣）** | 自分の砦に戻ると **体力が少しずつ回復**。砦には **耐久ゲージ** があり、**敵に砦を破壊されると敗北**。逆に **敵の砦を破壊すればクリア**。 |

## マップ / カメラ

戦場は画面（960×600）よりかなり広く（**1920×1200**）、**カメラが自分を追ってスクロール**します。

## ダイナマイト（砦破壊兵器）

`X` で設置する高威力兵器。**3秒後に爆発**し、特に砦へ大ダメージ。ただし **HPがあり、敵は爆発前に撃って解除**できる。自陣の砦に置かれたら撃ち落として防げ。設置中の砦が攻撃を受けると**画面に警告**が出ます。

## ビジュアル（Blender 製スプライト）

兵士・砦は **Blender でレンダリングした 3D スプライト**（`assets/*.png`、生成スクリプトは `tools/render_sprites.py`）。画像が無い環境では自動的にキャンバス描画にフォールバックします。

再生成：
```bash
blender --background --python tools/render_sprites.py
```

## BGM

6セクション・約40小節のアレンジ（イントロ→Aメロ→Bメロ→サビ→ブリッジ→リフト）で、単調にならないよう展開を付けています。

## 弾薬

マガジン10発。撃ち切ると3秒リロード（撃てない）だが、**撃つのをやめれば弾は自然回復**。乱射より制御射撃が有利。

味方・敵のCPも**距離に応じて武器を切り替え**（遠=スナイパー/近=ショットガン等）、**有効なアイテムだけ賢く拾う**（満タンなら回復は取らない）。試合が膠着した場合は自動で決着へ向かう仕組み（突撃→最終的に砦が削れる）も入っているので、プレイヤー不在でも進行が止まりません。

**6 vs 6（計12人）** で戦い、各チームは **役割（守備／攻撃／側面）と戦況に応じた作戦（守る／押す／均衡）** で連携します。砦は**山の上**に建ち、キャラは**歩行アニメ**付き。

## サウンド

BGM・効果音つき（Web Audio で生成、外部ファイル不要）。右上の 🔊 ボタンでミュート切替。

## キャラ選択・情報戦・objectives

- **キャラ選択**：開始時に青チーム6人から操作キャラを選択（選んだ枠の武器で出撃）。その兵士の視点でカメラが追従。
- **名前タグ**：頭上にチーム色の名前。**森に隠れた敵の名前は見えない**（情報戦）。
- **ミニマップ**：右上に全体図。味方・砦・宝箱・オアシス・自分の視界範囲を表示（隠れている敵は出ない）。
- **宝箱**：要所に出現。開けると一定時間の特殊武器 — 🔥火炎放射器（岩も焼く）/ 貫通ライフル / 破岩砲。
- **制圧拠点**：中央の中立拠点に（競合なしで）一定時間とどまると確保。確保すると**回復ゾーン**になり前線の足場になる。
- **砂漠＆オアシス**：砂地は移動が遅い。中立の**オアシスは回復ポイント**で取り合いに。
- **ステージは全10面**。後半（7面〜）は敵が**集中砲火などの連携**を見せる。

## アイテム（破壊した岩からドロップ）

| アイテム | 効果 |
|---|---|
| **HEAL** | 体力を回復 |
| **SPEED** | 移動速度アップ |
| **B.SPD** | 弾のスピードアップ |
| **RANGE** | 弾の飛距離アップ |
| **BOMB+** | 同時に置ける爆弾の数アップ |

## 爆弾（ボンバーマン風）

- `E` キーで足元に設置。少し待つと爆発します。
- 爆風は **岩を一気に破壊** し、範囲内のユニットに大ダメージ。
- **味方や自分も巻き込む** ので、設置したら離れること！

## LAN対戦（同じWi-Fiで同時対戦・最大12人）

同じWi-Fiの**1台のPC**でサーバーを起動し、各スマホ/PCがブラウザで参加します（最大12人＝青6/赤6、空き枠はAIが操作）。

```bash
# サーバーを動かすPCで（同じWi-Fi）
npm install        # WebSocket ライブラリ(ws)を入れる（初回のみ）
node server.js     # ポート8080で起動
```

- そのPCの表示する `http://localhost:8080/netclient.html` をホストが開く。
- 他の人は同じWi-Fiで **`http://<そのPCのIPアドレス>:8080/netclient.html`** を開く（IPは `ipconfig`/`ifconfig` で確認）。
- ロビーで名前を入れ、**青/赤の空きスロット（キャラ）を選んで参加**。途中参加・AIからの乗っ取りもOK。
- サーバーが権威（本体）として全員の試合を進め、各端末は入力送信＋描画のみ。

> ※ GitHub Pages（静的公開）はサーバーを動かせないため、LAN対戦には使えません。LAN対戦は上記のローカルサーバー方式です。インターネット越しの対戦は、サーバーをクラウド（Render/Fly 等）に置けば同じ仕組みで可能です。
