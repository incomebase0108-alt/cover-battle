# Cover Battle — 岩と森の 4 vs 4 バトル

ブラウザだけで遊べる、見下ろし型の 4 対 4 タクティカルバトルです。
岩を盾にし、森に隠れながら、敵チームを全滅させてステージを攻略しよう。
（スプラトゥーンのような「地形を使った撃ち合い」をシンプルにしたゲームです）

## 遊び方

`index.html` をブラウザで開くだけ。インストールやビルドは不要です。

```bash
# そのままダブルクリックで開くか、ローカルサーバーでもOK
python3 -m http.server 8000
# → http://localhost:8000 を開く
```

## 操作

| 操作 | キー |
|---|---|
| 移動 | `W` `A` `S` `D` または 矢印キー |
| 照準 | マウス |
| 射撃 | 左クリック または スペース |
| 爆弾設置 | `E`（または `Q`） |
| ロックオン 切替 | `R` |
| ロック対象 変更 | `Tab` |
| 武器選択 | `1` `2` `3` `4`（直接） / `F`（循環） |

### 武器

| 武器 | 特徴 |
|---|---|
| ライフル | 標準。10発/リロード3秒のバランス型 |
| スナイパー | 高威力・低連射・高弾速・長射程・装弾5発 |
| ショットガン | 1発で6粒拡散。近距離特化・射程短め |
| サブマシンガン | 超連射・装弾25発・威力低め・やや拡散 |

### スマホ操作

左の**バーチャルスティック**で移動（**斜め移動OK**・アナログ）、右のボタンで **撃／💣爆弾／🔒ロック／⇄対象変更**。タッチ端末では自動でロックオンON。

### 弾薬とリロード（緊迫感）

マガジンは **10発**。撃ち切ると **3秒のリロード**に入り、その間は撃てません。撃ちっぱなしにせず、残弾と間合いを管理するのが鍵。AIも同じルールで戦います（リロードの隙が攻めどき）。

## ルール

- **青チーム（4人）** を率いて **赤チーム（4人）** と戦う。あなたは青の1人を操作し、残りの味方3人はAIが担当します。
- 相手チームを **全滅** させるとステージクリア。味方が全滅すると敗北。
- ステージは全3面。クリアするごとに敵が強くなります。

## 地形

| 地形 | 効果 |
|---|---|
| 🪨 **岩** | 弾と移動をブロックする盾。撃ち続けると **破壊でき**、壊すと **アイテム** が出ることがある。 |
| ⛰️ **山** | 壊せない遮蔽。弾・視線・移動をブロックする。 |
| 🌲 **森** | 中に隠れると **敵からは見えなくなり狙われない**（味方と自分には見える）。敵が一定距離まで近づくと見つかる。 |
| 🌊 **川** | 通れるが **移動が遅くなる**。渡るときが危険。 |
| 🏰 **砦（自陣）** | 自分の砦に戻ると **体力が少しずつ回復**。砦には **耐久ゲージ** があり、**敵に砦を破壊されると敗北**。逆に **敵の砦を破壊すればクリア**。 |

## マップ / カメラ

戦場は画面（960×600）より広く（1536×960）、**カメラが自分を追ってスクロール**します。

## 弾薬

マガジン10発。撃ち切ると3秒リロード（撃てない）だが、**撃つのをやめれば弾は自然回復**。乱射より制御射撃が有利。

## サウンド

BGM・効果音つき（Web Audio で生成、外部ファイル不要）。右上の 🔊 ボタンでミュート切替。

## アイテム（破壊した岩からドロップ）

| アイテム | 効果 |
|---|---|
| **HEAL** | 体力を回復 |
| **SPEED** | 移動速度アップ |
| **B.SPD** | 弾のスピードアップ |
| **RANGE** | 弾の飛距離アップ |
| **BOMB+** | 同時に置ける爆弾の数アップ |

## 爆弾（ボンバーマン風）

- `E` キーで足元に設置。少し待つと爆発します。
- 爆風は **岩を一気に破壊** し、範囲内のユニットに大ダメージ。
- **味方や自分も巻き込む** ので、設置したら離れること！

| `js/input.js` | キーボード・マウス・タッチ入力 |
| `js/map.js` | 地形（岩・山・森・川・砦）の当たり判定と描画 |
| `js/entities.js` | ユニットと弾 |
| `js/ai.js` | 味方・敵AIの行動 |
| `js/game.js` | ゲームループ・状態管理・勝敗判定 |
| `js/ui.js` | HUD・メニューの表示制御 |
| `js/main.js` | 起動とボタン配線 |
| `tests/` | Node 製の自動テスト（`node tests/run.js` で実行） |

## デプロイ（外部サーバーへの公開）

静的サイトなのでどこにでも置けます。手順は [`DEPLOY.md`](DEPLOY.md) を参照。
`main` に push すると GitHub Actions（`.github/workflows/deploy-pages.yml`）で
GitHub Pages に自動公開されます（Settings → Pages → Source を「GitHub Actions」に設定）。
