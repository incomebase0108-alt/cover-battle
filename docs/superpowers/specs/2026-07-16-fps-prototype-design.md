# 3Dサバイバル化 前段：スマホfps検証プロトタイプ 設計書

作成: 2026-07-16（4号機 INCOMEBASE04）

## 目的

cover-battle を「限定マップの3Dサバイバル」へ拡張する方針（道①＝ブラウザのまま Three.js で3D化）の
最初の関門である **スマホ実機で8体同時に動かしたときの fps** を実測する。
キャラの本モデル（1号機担当）を待たず、棒人間で負荷だけ先に検証する。

## 成功基準

- スマホ実機（ブラウザ）で城下マップ＋棒人間8体が動き、画面上で fps が読める
- トグル操作で「影 / 街の密度 / 歩行アニメ」それぞれの負荷影響を切り分けられる
- 結果（60fps 維持可否・主犯）をワークログに記録できる

## 方針（承認済みの選択）

| 論点 | 決定 |
|---|---|
| 操作方式 | 1体を仮想ジョイスティックで操作＋7体はAI徘徊 |
| 負荷切り分け | 画面上トグル付き（影 ON/OFF・密度 低/中/高・歩行アニメ ON/OFF） |
| 置き場所 | `prototype/3d/`（本体 `js/` には一切触れない） |
| 実装アプローチ | 案A＝machi-maker の JS をコピーして土台にし、プロトタイプ専用コードを新規追加 |

案B（街データを事前JSON焼き込み・描画は新規最小実装）は、密度トグルのたびに
3パターン焼き込みが必要になりトグル方針と相性が悪いため不採用。

## 構成ファイル（すべて prototype/3d/ 配下）

| ファイル | 役割 | 出自 |
|---|---|---|
| `index.html` | 入口。Three.js r128 CDN 読込＋UI（fps HUD・トグル・ジョイスティック） | 新規 |
| `citygen.js` | 戦国城下町の生成（`era:'sengoku'`） | machi-maker からコピー（無改変） |
| `proto-viewer.js` | 描画層。InstancedMesh・タッチ回転。machi-maker viewer.js を縮めたもの | 改変コピー |
| `stickman.js` | 棒人間（頭 sphere＋胴・手足 cyl）。青4体・赤4体に色分け、歩行の手足振り | 新規 |
| `bots.js` | 7体のAI徘徊。ランダム目標地点へ2D平面移動（座標系は平面のまま＝本番と同じ前提） | 新規 |
| `joystick.js` | 仮想ジョイスティック（左親指=移動、画面ドラッグ=視点）。ライブラリなし自作 | 新規 |
| `main.js` | 組み立て＋fps計測（直近1秒の平均/最低 fps を HUD 表示） | 新規 |

- Three.js は machi-maker と同じ **r128 を CDN 読込**（スマホ検証は LAN 配信なので CDN 可）。
- 街データは実行時に `CityGen.generate({era:'sengoku', ...})` で生成。密度トグルはパラメータを変えて再生成する。

## トグル仕様（画面上のボタン）

| トグル | 切替内容 | 目的 |
|---|---|---|
| 影 ON/OFF | `renderer.shadowMap.enabled`＋DirectionalLight の castShadow | 主犯候補①の切り分け |
| 密度 低/中/高 | citygen パラメータ（blocks/density 等）を変えて街を再生成 | 環境描画コストの切り分け |
| 歩行アニメ ON/OFF | 棒人間の手足振り更新を停止 | キャラアニメコストの切り分け |

## 検証手順

1. PC で `node server.js`（既存 LAN サーバ。port 8080 の静的配信をそのまま使う）
2. スマホで `http://<PC-IP>:8080/prototype/3d/index.html` を開く
3. ジョイスティックで操作しながら HUD の fps を読み、トグルで負荷源を特定
4. 結果をワークログ（.claude/worklog/CURRENT.md）に記録

PC 側の描画事前確認は Edge ヘッドレスのスクショで行う（お城メーカーで確立済みの手順）。
fps の実測値はスマホ実機で本人に読んでもらう。

## やらないこと（YAGNI）

- cover-battle 本体（js/・server.js）の変更
- ゲームロジック（戦闘・当たり判定）の3D移植 — 今回は描画負荷の検証のみ
- キャラの本モデル組込（1号機担当。棒人間で代用）
- ネット同期（8体はローカルで動かす。サーバ権威型の負荷は本番フェーズで検証）

## エラー・例外の扱い

- WebGL 未対応端末では「WebGL が使えません」と表示して終了（フォールバック描画は作らない）
- citygen 生成失敗時はブラウザ console にエラーを出す（プロトタイプなので握りつぶさない）

## テスト

- 自動テストは書かない（使い捨て検証プロトタイプのため）
- 代わりに Edge ヘッドレススクショで「街＋棒人間8体が描画されている」ことを確認してから実機へ
